# **Bayesian Inference for Agentic Evaluation of Task Difficulty**

## **1 Introduction, Settings, & Motivation**

The core objective of this project is to derive calibrated estimates of **task difficulty** ($$\\beta$$) for **long-horizon** **agents** based on **human-interpretable** features of the tasks and agents, in two settings:

1. **The Solvable Regime:** For tasks that are solvable by current models, running full evaluations is computationally expensive. Can we accurately predict difficulty in a *cheaper* way than running the agent evaluation itself? In particular, we will use only the problem statement and environment.   
   1. Motivation: these cheap difficulty predictions can be used to sample problems of a useful difficulty during RL, avoiding wasted rollouts on too hard or too easy problems.   
2. **The Frontier Regime:** For tasks beyond current frontier model capabilities, we currently lack signal about precisely how hard they are. Can we estimate *how far* these tasks are from being solved, or understand *what* about agents’ interactions with these tasks makes them so difficult?   
   1.  Motivation 1: We can use these estimates to **forecast** when in the future frontier models will be able to solve specific difficult tasks within the dataset with 50% probability. See [Experiment D](#3.4-experiment-d:-time-horizon-prediction-—-completed) for more details.   
   2. Motivation 2:  Understanding how failure modes relate to difficulty of problems is useful for improving agents. 

In answering these questions, we hope to improve our understanding of what makes a problem difficult, and what signals we get about this from an agent’s failing attempt at solving the problem. It would be especially useful if these signals generalized across datasets.

## **2 Theoretical Framework: Amortized Bayesian Inference**

We will use the IRT framework that every task $$i$$ has a latent difficulty $$\\beta\_i$$. Our main goal throughout will be to provide good estimates of $$\\beta\_i$$ with various affordances. We use the standard IRT model of success probability of an agent $$j$$ on task $$i$$: 

$$P(y\_{ij} \= 1 \\mid \\theta\_j, \\alpha\_i, \\beta\_i) \= \\sigma(\\alpha\_i(\\theta\_j \- \\beta\_i))$$

To address the solvable regime setting, we will train one linear model that predicts a prior estimate of this latent difficulty just using textual data and metadata of the problem and access to the problem environment from Lunette. [Truong et al. (2025)](https://arxiv.org/pdf/2503.13335) basically did this but for Q\&A benchmarks and without environment access (since no such thing exists).  

To address the frontier regime setting for when we observe a new problem \+ failing trajectory data, we train a linear model to predict the update to our prior difficulty estimates. 

### 2.2 Goal 1: The No Trajectory Prior

**Objective:** Predict task difficulty using only static features.

* **Input ($$x\_i$$):** Using Lunette GradingPlan as a more powerful LLM as judge on the problem \+ environment. For her project, Daria will be using embeddings of the problem text \+ metadata for this $$x\_i$$.   
* **Model**: A linear model predicting difficulty $$\\beta\_i$$ from problem input features $$x\_i$$.   
* **Training:** Train the IRT ability parameters and the linear model parameters to maximize log likelihood of the observed successes/failures (Truong et al 2025).  
* **Inference:** Lunette tells us $$x\_i$$ on new question, linear model predicts difficulty $$\\beta\_i$$ (**without having to relearn the IRT parameters\!**), IRT predicts the probability that that agent gets that problem correct.

### 2.3 Goal 2: The Failure-Informed Posterior (Weak to Strong generalization)

This is the most tenuous / critical part of the proposal. Advice/Critique here would be appreciated.

**Objective:** Refine the difficulty estimate of really hard problems using failure trajectory analysis from agents too weak to solve the problems.

The key idea will be that instead of training on problems that are too hard for current-day frontier models to solve (which we don’t have good labels of actual difficulty for), we’ll restrict ourselves to only using models from 6 months ago for the trajectories, but still use problems that some current-day models can solve so that we know the true difficulties. 

* **Input:**  
  * The frozen Prior difficulty estimate $$\\hat{\\beta}\_{prior} \= \\mu\_\\phi(x\_i)$$  
  * A set of failure trajectories $$\\{\\tau\_{ij}\\}\_{j \\in M}$$ generated by the weak models $$j$$ on task $$i$$.  
* **Model**:  
  * For each trajectory, we will also use Lunette Grading to get a feature vector of information about the trajectory, $$f(\\tau\_{ij})$$, and a set of [failure modes](https://arxiv.org/pdf/2509.13941) $$c(\\tau\_{ij})$$ of the trajectory. See [section 2.4.2](#heading=h.mqar133f4su3) for more details. Daria will once again use embeddings as her function $$f$$, and can use a classification head on those embeddings as $$c$$.   
  * The posterior difficulty estimate is the prior plus a learned correction term parametrized by a set of linear models, one for each failure mode $$c$$, with weights $$\\psi\_c$$, i.e.   
    $$\\mu\_{\\phi}(x\_i) \+\\left( \\frac{1}{|M|}\\sum\_{j \\in M} \\frac{1}{|c(\\tau\_{ij})|} \\sum\_{k \\in c(\\tau\_{ij})} \\psi\_{k}^Tf(\\tau\_{ij})\\right)$$   
    where $$M$$ is the set of models for which we have inference time trajectories for the task   
* **Training:**   
1. **Data Selection:** We will construct a training set $$\\mathcal{D}\_{train}$$ of tasks where the oldest models (e.g. \> 1 year ago) all have a success rate 0-20%. But since our use case is to use our model predictions to forecast the future, for the validation dataset $$D\_{valid}$$, we should pick tasks where the somewhat less old models (e.g. 6 months \- 1 year ago) have success rate 0-20%. We will test if the weights $$\\psi$$ trained on $$D\_{train}$$ transfer over to $$D\_{valid}$$.   
2. **The Objective Function**: We freeze the prior model $$\\mu\_\\phi$$ and train parameters $$\\psi$$ to minimize the regularized L2 loss. The loss function $$\\mathcal{L}(\\psi)$$ is 

$$\\mathcal{L}(\\psi)=\\frac{1}{|\\mathcal{D}\_{\\text{train}}|}\\sum\_{i \\in \\mathcal{D}\_{\\text{train}}}\\left\\| \\underbrace{\\left( \\mu\_\\phi(x\_i) \+ \\left( \\frac{1}{|M|}\\sum\_{j \\in M} \\frac{1}{|c(\\tau\_{ij})|} \\sum\_{k \\in c(\\tau\_{ij})} \\psi\_{k}^Tf(\\tau\_{ij})\\right)\\right)}\_{\\text{Predicted Posterior}} \- \\underbrace{\\beta\_i^{GT}}\_{\\text{True Difficulty}} \\right\\|^2 \\;+\\; \\lambda \\, \\|\\psi\\|\_2^2 $$

## **3 Experimental Plan**

We focus on two core experiments to validate the Prior (Goal/Regime 1\) and the Posterior (Goal/Regime 2), along with some other experiments to explore questions we have that we get sort of for free from this work. 

### 3.1 Experiment A: Prior Validation (IRT AUC)

Run the setup in Section 2.2. 

* **Evaluate** AUC on a held-out test set of tasks to see if we can predict agents' pass@k success probability zero-shot.   
* Use as **baseline** just predicting that agent’s average performance, and compare Lunette Grading, LLM as judge, and embeddings, with and without solution access. 

### 3.2 Experiment B: Posterior Validation (IRT Difficulty Prediction for Harder Tasks) 

Run the setup in Section 2.3. 

* **Evaluate** correlation **($$r$$)** between Predicted Difficulty and Actual Strong Model IRT Difficulty score on $$D\_{valid}$$.   
  * also get a scatterplot of our predicted difficulties against the actual IRT difficulty scores  
* Use as **baseline** the Prior model only. 

Extra plots / subexperiments: 

* retroactively forecast model arrival dates for models from the last 6 months  
* change the agent trajectories used for each task and see if difficulty predictions vary wildly.   
  * and then the coefficients on the Lunette judge features tell us human understandable features that make the problem hard for models in general   
* Do agents behave differently when they are close to being smart enough to solve a problem versus when they are hopelessly lost?   
  * Look at the failing trajectories on the easier impossible problems vs the harder impossible problems, and see if a 2D PCA or t-SNE visualization of the encodings of trajectories are different for the two groups.
